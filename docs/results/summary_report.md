# AI量化LLM大横评 — 综合报告

## 1. 项目概述

本项目对 **13 个主流大语言模型**在量化交易场景下的综合能力进行了标准化横向评测。测评覆盖 **4 大维度**（计算能力、分析能力、规则执行+约束遵循、数据处理+长上下文），外加 **JSON 格式合规性**作为贯穿维度，共计 **987 次 API 调用**。

> 注 3：**GLM 5.2 为主测完成后补测**，经智谱官方 BigModel API（一方端点）接入，与经 OpenRouter 的 GLM 5.1 渠道不同。条件：`temperature=0`、`reasoning_effort=max`、`max_tokens=49152`、**3 轮独立采样**（其余模型 T2/T4/T5 为 1 轮）。各轮真实独立（非缓存），可信度高于经本地代理的 Fable 5；费用按官方价（$1.4/$4.4 per 1M）折算，由新账号 20M 免费额度覆盖。

## 3. 能力画像分析

### 3.1 各模型能力雷达

**Claude Fable 5 — 新一代断层冠军，首个"无明显短板"的模型**
- 综合 81.5%（第 1，首个突破 80%），T1 计算 83.3% 断层第一，T5 满分（零误报零漏报）
- 给公式模式下三个指标 27/27 全对（含 MACD 40 步递推），三个数据段得分零波动
- 思考型模型：按任务难度自适应分配推理预算（T2 平均 62 秒深度思考 vs T5 仅 9-13 秒）
- 弱点：贵（折算 $12.85 全场最高，官方单价 $10/$50 是 Opus 4.8 的 2 倍）；T2 结构分析仍未破行业天花板（49.1%）；保留了 Claude 家族的代码块包裹习惯（T4 JSON 72.5%）
- 成绩可信度：已通过五步审计复核（脚本零偏差 + 官方端点 4/4 逐字复现），见技术报告第 12 章

**GPT-5.5 — 公开渠道可得的最强均衡型选手**
- 综合 73.9%（第 2，OpenRouter 直接可用的模型中第 1），规则和数据满分
- 计算中等偏上（46.3%），分析中等偏上（49.4%）
- JSON 合规全程 100%
- 弱点：贵（实测费用 $4.08，公开渠道最高）

**Claude Opus 4.8 — 旧系列计算最强，高端全能**
- 旧 11 模型中唯一 T1 超过 50% 的模型（53.7%），RSI 计算全对
- 规则执行和数据处理也在高位（94.1% / 90.9%）
- 弱点：响应速度快但 T2 分析不突出（46.2%）

**GLM 5.1 — 性价比之王**
- 总费用仅 $0.96，综合排名第 4
- T4 规则执行 96.9%（第 3 名），T5 数据处理 93.9%
- T1 计算 48.1%（第 4 名），SMA 不给公式模式全对
- 弱点：JSON 合规最低档（73.6%），T2 分析能力弱（40.0%）

**DeepSeek V4 Pro — 高性价比推理型**
- 总费用仅 $0.45，JSON 合规 100%
- T5 数据处理 97.0%（第 6 名）
- 弱点：高度依赖明确规则（T4 给规则 100%，不给规则 71.5%）

**Gemini 3.1 Pro — 极端分化**
- T2 分析第 1（52.4%），T4 规则满分
- 但 T5 的 500 根数据崩溃到 0%（总分仅 66.7%），T1 倒数
- 不适合长数据处理任务

**Qwen3.7 Max — 分析强但计算极弱**
- T2 分析第 2（50.9%），T5 数据处理满分
- 但 T1 计算仅 3.7%（垫底），几乎不会算任何指标
- 适合分析和检测类任务，不适合需要精确数值的场景

**Kimi K2.6 — 数据处理满分但慢**
- T5 满分，T4 规则 93.8%
- 但 T1 仅 24.1%，T2 仅 39.2%（垫底）
- 响应速度全场最慢（T2 多个用例超过 10 分钟）

## 6. 核心结论

### 结论一：Claude Fable 5 综合登顶（81.5%），GLM 5.2 季军反超 Opus 4.8，GPT-5.5 仍是公开渠道最强

补测的 Claude Fable 5 以 81.5% 成为首个突破 80% 的模型：T1 计算断层第一（83.3%）、T5 满分、全部维度进前五。其计算成绩经五步审计独立复核为模型真实能力（官方端点逐字复现）。但它经由本地代理补测（按官方定价折算 $12.85 全场最贵），测试条件与主测不完全一致（见局限性）。**国产新旗舰 GLM 5.2 综合 72.7% 跃居第 3，反超 Claude Opus 4.8**——计算 61.1%（全场第 2）拉动，经智谱一方 API、temperature=0、3 轮独立采样取得（渠道比 Fable 5 更干净）；GPT-5.5（73.9%）仍是公开 OpenRouter 渠道综合最强，但 GLM 5.2 已贴身紧追、成本仅其零头。

在 OpenRouter 可直接接入的模型中，GPT-5.5（73.9%）仍是综合最强：T4/T5 双满分、JSON 全程 100%。但总费用 $4.08 是 DeepSeek 的 9 倍、GLM 的 4 倍。如果预算充足且追求最高可靠性，新一代旗舰是首选；否则国产替代方案更具性价比。

### 结论二：国产模型在"执行类"任务上已达到御三家水平

在 T4（规则执行）和 T5（数据处理）两个维度上，国产模型的表现已经追平甚至超越御三家（本结论按主测 11 模型口径，不含补测的 Fable 5）：
- GLM 5.1 T4 得分 96.9%（第 3），MiniMax M3 / Kimi K2.6 / Qwen3.7 Max T5 满分
- 国产模型在这两个维度的平均得分（92.8%）与御三家（92.4%）几乎相同
- 但在 T1（计算）和 T2（分析）上，国产模型平均 33.5%，御三家平均 46.7%，差距仍在
- 补测的 Fable 5 进一步拉开了 T1 的代差（83.3% vs 国产最高 48.1%），"执行类已追平、认知类有代差"的格局更加明显

### 结论三："没有全能冠军"首次松动——但维度洗牌依旧剧烈

| 模型 | 最强维度 | 最弱维度 | 极差 |
|------|---------|---------|------|
| Claude Fable 5 | T5 (100%) | T2 (49.1%) | 50.9% |
| GPT-5.5 | T4/T5 (100%) | T1 (46.3%) | 53.7% |
| Claude Opus 4.8 | T1 (53.7%) | T2 (46.2%) | 47.5% |
| Gemini 3.1 Pro | T4 (100%) | T1 (38.9%) | 61.1% |
| Qwen3.7 Max | T5 (100%) | T1 (3.7%) | **96.3%** |
| GLM 5.1 | T4 (96.9%) | T2 (40.0%) | 56.9% |

Qwen3.7 Max 的极差达到 96.3%——T5 满分但 T1 几乎为零。补测的 Claude Fable 5 是第一个在全部维度都进前五的模型，让"单模型兜底"第一次变得可行；但它的极差仍有 50.9%（T2 仍是短板），且排名在维度间照样洗牌（T1 第 1 → T4 并列第 5）。在量化 Agent 设计中，**按任务类型选择最合适的模型依然是最优解**。

### 结论四：计算瓶颈已被新一代攻破，结构分析瓶颈依旧

| 维度 | 全模型平均 | 最高分 |
|------|-----------|--------|
| T1 计算 | 40.0% | **83.3%（Fable 5）** |
| T2 分析 | 45.7% | 52.4%（Gemini） |
| T4 规则 | 92.2% | 100% |
| T5 数据 | 93.7% | 100% |

主测时 T1/T2 最高分都没超过 54%，我们曾认为两者都是"根本性能力缺陷"。补测修正了一半：**Claude Fable 5 把 T1 拉到 83.3%（给公式 27/27 全对，含 MACD 40 步递推），证明精确计算是会随代际进化解决的工程问题**。但 T2 连最新模型也停在 49.1%——多 K 线结构推理仍是全行业未破的认知瓶颈。

实践建议：指标计算仍首选传统代码（确定性 + 零成本），Fable 5 可作为"必须用 LLM 时"的唯一兜底（必须显式给公式）；结构分析则不应依赖任何 LLM。

### 结论五：JSON 合规性决定了 Agent 协作的可靠性

GPT-5.5 和 DeepSeek V4 Pro 全程 JSON 合规 100%，意味着它们的输出可以直接 `json.loads()` 解析。而 GLM 5.1 仅 73.6%，接入 Agent 框架时需要额外的输出清洗层。值得注意的是综合冠军 Fable 5 也只有 85.9%——Claude 家族的代码块包裹习惯跨代未改，得分高不等于工程友好。

在多 Agent 协作场景中，JSON 合规性是基础设施级的要求。**推荐使用 GPT-5.5 或 DeepSeek V4 Pro 作为 Agent 间数据交换的模型**。

### 结论六：量化 Agent 的最佳模型组合建议

基于全部测评数据，针对不同场景推荐：

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 指标计算 | 传统代码优先；LLM 方案仅 **Claude Fable 5**（必须给公式） | Fable 5 给公式 27/27 全对，其余模型最高仅 53.7% |
| K线分析/SMC | Gemini 3.1 Pro 或 Qwen3.7 Max | T2 前两名，但需控制数据量 |
| 交易信号判定 | GPT-5.5 或 GLM 5.1 | T4 满分/近满分 |
| 数据质量检测 | GPT-5.5 / Claude Fable 5 / MiniMax M3 / Kimi K2.6 | T5 满分 |
| Agent 数据交换 | DeepSeek V4 Pro | JSON 100% + 低成本 |
| 预算极其有限 | **MiMo V2.5 Pro** | 总费用 $0.28，综合 61.2% |
| 综合性价比最优 | **GLM 5.1** | $0.96 拿到 69.7%（第 4 名） |
| 追求最高性能 | **Claude Fable 5**（81.5%）；仅限公开 API 则 **GPT-5.5**（73.9%） | 综合前两名 |

## 7. 测评局限性与改进方向

1. **T1 数据部分丢失**：T1 的详细 per-call 数据因落盘 bug 丢失，仅保留了 summary 级别的分数。后续测评已修复此问题。

2. **每个用例仅 1 轮（T2/T4/T5）**：为节省成本，T2 及之后的测评仅跑 1 轮，无法评估稳定性。T1 的 3 轮数据显示部分模型稳定性差异显著。

3. **异常检测的异常过于明显**：T5 中注入的异常（价格骤降 90%、负成交量）对 LLM 来说容易识别，导致 5 个模型满分。未来可设计更微妙的异常（2-3% 偏差）来增加区分度。

4. **MiniMax 版本不一致**：T1 使用 M2.7，T2 及之后使用 M3。综合排名中合并了两个版本的成绩（T1 取 M2.7，T2/T4/T5 取 M3），可能不完全反映单一版本的真实水平。

5. **单一数据源**：所有测评使用 BTC/USDT 1小时 K线，未覆盖其他币种、时间周期和市场状态（如极端行情）。

6. **缺少多轮对话测评**：当前测评均为单轮问答，未测试模型在多轮 Agent 交互中的上下文保持和任务连贯性。

7. **Claude Fable 5 补测条件不完全一致，但成绩已经独立审计复核**：① 经本地 Anthropic 中转代理（账号池架构）接入，其余模型走 OpenRouter；② 该模型拒绝 `temperature` 参数，使用默认采样（其余模型为 0.0）；③ 显式设置 `max_tokens=32000`（代理默认 8192 曾导致重型用例空响应/截断，归档后重跑修复，原始失败记录保存在 `results/_failed_rounds/`）；④ 代理每次调用注入约 515 token 开销（实测），token 统计偏高约 3%；⑤ 代理疑似响应缓存——三轮响应逐字节相同，其三轮通过率不构成独立稳定性证据；⑥ 费用按官方定价（$10/$50 per 1M）折算。**针对"成绩好得反常"的怀疑，已执行五步审计**（管线重评分、误差指纹、跨轮一致性、Decimal 舍入取证、官方端点对照），结论：脚本无错、能力为真（官方端点 4/4 逐字复现，SMA 精确到第三位小数）；渠道问题仅影响 token 统计与轮间独立性。完整方法与证据见技术报告第 12 章。
