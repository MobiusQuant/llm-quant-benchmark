# T5+T6 数据处理 + 长上下文能力测评报告

## 1. 测评概述

本测评旨在评估主流大语言模型在量化交易场景下的**数据质量检测能力**和**长上下文保持能力**。

在真实K线数据中预埋5种已知异常，分别在50根、200根、500根三种数据长度下，要求模型找出所有异常并正确分类。通过对比不同数据量下的检出准确率，同时衡量模型的数据处理能力（T5）和长上下文衰减特性（T6）。

成本：约 $1.25（OpenRouter 11 模型）+ ~$0.41（Claude Fable 5 补测）+ ~$0.31（GLM 5.2 补测，智谱一方 API，3 轮，按官方价折算）

### 测评模型

共 13 个模型，覆盖 3 家海外厂商 + 6 家国产厂商：

| 分类 | 模型 |
|------|------|
| 御三家旗舰 | GPT-5.5、Claude Opus 4.8、Gemini 3.1 Pro |
| 御三家次旗舰 | Claude Opus 4.6、Claude Sonnet 4.6 |
| 国产第一梯队 | DeepSeek V4 Pro、Qwen3.7 Max、Kimi K2.6、GLM 5.1、MiniMax M3、MiMo V2.5 Pro |
| 后续补测 | Claude Fable 5（Anthropic 2026-06 最新旗舰）、GLM 5.2（智谱最新旗舰，一方 BigModel API） |

> **Claude Fable 5 补测条件说明**：经本地 Anthropic 中转代理接入；模型拒绝 `temperature` 参数（使用默认采样，其余模型为 0.0）；费用按官方定价（$10/$50 per 1M）折算。其成绩已经五步审计独立复核，见 T1 报告第 7 节与技术报告第 12 章。
>
> **GLM 5.2 补测条件说明**：经智谱官方 BigModel API（一方端点）接入，与 GLM 5.1（OpenRouter）渠道不同；`temperature=0`、`reasoning_effort=max`、**3 轮独立采样**（其余模型 T5 为 1 轮）。费用按官方价（$1.4/$4.4 per 1M）折算，由 20M 免费额度覆盖。详见 T1 报告与技术报告第 12 章。

### 测评规模

- 3 个测试用例（50根 / 200根 / 500根）× 1 轮 × 12 个模型 + 3 × 3 轮（GLM 5.2）= **45 次 API 调用**
- 同步评测 JSON 格式合规性

## 2. 测评设计

### 2.1 异常类型

在每段数据中预埋 5 种异常，均匀分布在头部、中部、尾部：

| 异常类型 | 说明 | 注入方式 |
|---------|------|---------|
| missing_value | 缺失值 | 将某根 K 线的 close 置为空 |
| ohlc_logic_error | OHLC 逻辑错误 | 交换 high 和 low，使 high < low |
| duplicate_timestamp | 时间戳重复 | 复制上一根 K 线的时间戳 |
| price_spike | 价格异常 | 将 close 缩小为原值的 1/10 |
| volume_anomaly | 成交量异常 | 将 volume 设为 -500 |

### 2.2 数据长度与异常位置

| 数据长度 | 异常数量 | 异常位置（行号） |
|---------|---------|----------------|
| 50 根 | 5 个 | 第 9、17、25、33、41 行 |
| 200 根 | 5 个 | 第 34、67、100、133、166 行 |
| 500 根 | 5 个 | 第 84、167、250、333、416 行 |

三种长度使用相同的 5 种异常类型，位置按数据长度等比例调整，确保头部/中部/尾部均有覆盖。

### 2.3 数据来源

使用 BTC/USDT 1小时 K线真实行情数据（2024年1月初），分别截取前 50、200、500 根后注入异常。

### 2.4 评分标准

- **T5 数据处理得分**：F1 score = 2 × Precision × Recall / (Precision + Recall)
  - 匹配条件：行号（row）和异常类型（type）都正确
  - Precision = 正确检出 / 模型总报出数（衡量误报）
  - Recall = 正确检出 / 实际异常数（衡量漏报）
- **T6 长上下文得分**：对比 50根 → 200根 → 500根 的 F1 衰减趋势
- **JSON 合规性**：搭车评分

## 4. 分维度分析

### 4.1 T5 数据处理能力：Precision vs Recall

| 模型 | 平均 Precision | 平均 Recall | 错误倾向 |
|------|---------------|-------------|---------|
| GPT-5.5 | 1.00 | 1.00 | 无 |
| Claude Fable 5 | 1.00 | 1.00 | 无 |
| MiniMax M3 | 1.00 | 1.00 | 无 |
| Kimi K2.6 | 1.00 | 1.00 | 无 |
| Qwen3.7 Max | 1.00 | 1.00 | 无 |
| DeepSeek V4 Pro | 0.94 | 1.00 | 偶尔误报 |
| Claude Sonnet 4.6 | 0.89 | 1.00 | 偶尔误报 |
| GLM 5.1 | 0.89 | 1.00 | 偶尔误报 |
| Claude Opus 4.6 | 0.85 | 1.00 | 误报较多 |
| Claude Opus 4.8 | 0.83 | 1.00 | 稳定误报 |
| MiMo V2.5 Pro | 0.88 | 0.93 | 误报+漏报 |
| Gemini 3.1 Pro | 0.67 | 0.67 | 500根全部漏报 |

**核心发现：几乎所有模型都能找到全部异常（Recall≈100%），差异主要在误报率（Precision）。** 

5 个模型（GPT-5.5、Claude Fable 5、MiniMax M3、Kimi K2.6、Qwen3.7 Max）做到了零误报零漏报。Claude 旧系列和 GLM 倾向于多报 1-2 个不存在的异常。MiMo V2.5 Pro 是唯一在 50 根数据上漏报的模型（fn=1）。

### 4.2 T6 长上下文衰减分析

| 衰减模式 | 模型 | 说明 |
|---------|------|------|
| **无衰减** | GPT-5.5, Claude Fable 5, MiniMax M3, Kimi K2.6, Qwen3.7 Max, Claude Opus 4.8 | 3个长度得分一致 |
| **反向衰减（越长越好）** | Claude Sonnet 4.6, GLM 5.1, MiMo V2.5 Pro | 500根反而满分 |
| **轻微衰减** | DeepSeek V4 Pro, Claude Opus 4.6 | 500根略降 |
| **断崖式崩溃** | **Gemini 3.1 Pro** | 50/200根满分，500根直接0分 |

**预期的"数据越长准确率越低"并未普遍出现。** 多数模型在 500 根数据上表现等于甚至优于 50 根。可能原因：

1. 异常在 500 根数据中间距更大（每隔 ~83 行一个），比 50 根数据（每隔 ~8 行）更容易逐段分析
2. 5 个异常在 500 根数据中占比仅 1%，模型可能更容易将"正常"作为基线来识别偏差
3. 本次预埋的异常都非常明显（价格骤降 90%、负数成交量等），即使在长文本中也很容易发现

**Gemini 3.1 Pro 的断崖式崩溃是唯一的真正长上下文失败案例。** 50 根和 200 根都满分，但 500 根（约 35,000 字符）时 5 个异常一个都没检出。查看其响应延迟（296 秒），可能是在长输入时陷入了不同的处理模式，放弃了逐行扫描。

## 7. 关键发现

### 发现一：异常检测是所有测评中 LLM 表现最好的维度

5 个模型满分（100%，含补测的 Claude Fable 5），除 Gemini 外所有模型平均 F1 超过 90%。相比 T1（最高 83.3%）、T2（最高 52.4%）和 T4（最高 100%），T5 的整体水平最高。

异常检测本质上是"在规律中找不规律"——模型只需要识别偏离常态的数据点，而不需要复杂的数值计算或结构推理。这恰好是 LLM 模式匹配能力的强项。

### 发现二：Gemini 3.1 Pro 的 500 根断崖式崩溃

Gemini 在 50 根和 200 根上满分，500 根直接 0 分——5 个异常一个都没找到。这是本次全部测评中最极端的长上下文失败案例。响应时间从 45 秒暴增到 296 秒，说明模型在处理 ~35K 字符的输入时进入了不同的处理模式。

对于量化 Agent 而言，这意味着 **Gemini 3.1 Pro 不适合处理超过 200 根 K 线的数据分析任务**。

### 发现三：预期的长上下文衰减并未普遍出现

多数模型在 500 根数据上表现等于甚至优于 50 根。3 个模型（Claude Sonnet、GLM、MiMo）在 500 根时反而满分。

这可能有两个原因：
1. **异常过于明显**：价格突降 90%、负数成交量等异常在任何长度的数据中都很容易发现
2. **间距效应**：500 根数据中异常间距约 83 行，比 50 根数据（间距约 8 行）更容易逐段分析

要真正测试长上下文衰减，可能需要更微妙的异常（如 2-3% 的价格偏差），或者将异常集中放在数据中间/尾部。

### 发现四：误报是主要错误类型，漏报极为罕见

除 Gemini 500 根和 MiMo 50 根外，所有模型的 Recall 都是 100%。差异完全体现在 Precision 上——Claude 系列稳定多报 1 个误报，而 GPT-5.5 等 4 个模型做到了零误报。

在量化场景中，**漏报比误报更危险**（漏掉异常数据可能导致策略基于错误数据做出决策）。从这个角度看，几乎所有模型都足够可靠。

### 发现五：T1-T5 排名持续洗牌，没有全能冠军

| 模型 | T1 | T2 | T4 | T5 |
|------|----|----|----|----|
| Claude Fable 5 | **1** | 4 | 5 | **1** |
| GPT-5.5 | 5 | 3 | 1 | **1** |
| Claude Opus 4.8 | 2 | 6 | 4 | 10 |
| Gemini 3.1 Pro | 8 | 1 | 1 | **12** |
| Qwen3.7 Max | 12 | 2 | 10 | **1** |
| MiniMax M3 | — | 7 | 5 | **1** |
| Kimi K2.6 | 10 | 12 | 5 | **1** |

Gemini 3.1 Pro 在 T2 和 T4 都拿第一，但在 T5 跌到最后。Kimi K2.6 在 T2 垫底，T5 满分。每个模型都有自己的强项和弱项。补测的 Claude Fable 5 是第一个在所有维度都进前五的模型（T1 第 1、T2 第 4、T4 并列第 5、T5 并列第 1）——**"没有全能冠军"的结论首次出现松动，但 T2 的绝对分数（49.1%）仍然说明结构分析短板是全行业性的**。

### 发现六：MiniMax M3 是成本效益的持续赢家

MiniMax M3 以 $0.015 的成本拿到满分。在 T4 中性价比排名第 2，T5 中排名第 1。对于异常检测和规则执行类任务，MiniMax M3 是最具成本效益的选择。
