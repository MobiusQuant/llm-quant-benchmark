# T4 规则执行 + 约束遵循测评报告

## 1. 测评概述

本测评旨在评估主流大语言模型在量化交易 Agent 场景下的**规则执行能力**和**约束遵循能力**。

模拟一个量化交易 Skill 的调用场景：给模型一段 K线数据（附带预算好的指标值）、一套交易规则、一组行为约束，要求模型判断交易信号并按规范输出。一次调用同时评估三个维度：信号判定准确性（T4）、约束遵守率（T7）、JSON 格式合规性（T3）。

成本：约 $0.55（OpenRouter 11 模型）+ ~$0.23（Claude Fable 5 补测）+ ~$0.14（GLM 5.2 补测，智谱一方 API，3 轮，按官方价折算）

### 测评模型

共 13 个模型，覆盖 3 家海外厂商 + 6 家国产厂商：

| 分类 | 模型 |
|------|------|
| 御三家旗舰 | GPT-5.5、Claude Opus 4.8、Gemini 3.1 Pro |
| 御三家次旗舰 | Claude Opus 4.6、Claude Sonnet 4.6 |
| 国产第一梯队 | DeepSeek V4 Pro、Qwen3.7 Max、Kimi K2.6、GLM 5.1、MiniMax M3、MiMo V2.5 Pro |
| 后续补测 | Claude Fable 5（Anthropic 2026-06 最新旗舰）、GLM 5.2（智谱最新旗舰，一方 BigModel API） |

> **Claude Fable 5 补测条件说明**：经本地 Anthropic 中转代理接入；模型拒绝 `temperature` 参数（使用默认采样，其余模型为 0.0）；费用按官方定价（$10/$50 per 1M）折算。其成绩已经五步审计独立复核，见 T1 报告第 7 节与技术报告第 12 章。
>
> **GLM 5.2 补测条件说明**：经智谱官方 BigModel API（一方端点）接入，与 GLM 5.1（OpenRouter）渠道不同；`temperature=0`、`reasoning_effort=max`、**3 轮独立采样**（其余模型 T4 为 1 轮）。约束遵守经核验仍为 100%。费用按官方价（$1.4/$4.4 per 1M）折算，由 20M 免费额度覆盖。详见 T1 报告与技术报告第 12 章。

### 测评规模

- 4 个测试用例 × 1 轮 × 12 个模型 + 4 × 3 轮（GLM 5.2）= **60 次 API 调用**
- 同步评测 JSON 格式合规性

## 2. 测评设计

### 2.1 题目设计

2 个交易规则场景 × 2 种模式 = 4 个用例：

| 编号 | 规则复杂度 | 约束数量 | 说明 |
|------|-----------|---------|------|
| Q3 | OR + AND 组合条件 | 4 个约束 | 4 个条件组（A/B/C/D），OR+AND 组合判断 BUY/SELL/HOLD |
| Q4 | 嵌套条件 + 优先级 | 5 个约束 | 先判趋势方向（bullish/bearish/sideways），再基于趋势判断入场信号 |

### 2.2 交易规则详情

**Q3（OR + AND 组合）：**
- 条件组 A（BUY）：RSI < 35 且 MACD 金叉
- 条件组 B（BUY）：Close < BB 下轨 且 Close < EMA20
- 条件组 C（SELL）：RSI > 65 且 MACD 死叉
- 条件组 D（SELL）：Close > BB 上轨 且 Close > EMA20
- 逻辑：C 或 D → SELL；A 或 B → BUY；SELL 优先于 BUY

**Q4（嵌套条件）：**
- Step 1：基于最近 10 根 K线的 EMA20 变化判断趋势方向
- Step 2：基于趋势方向选择不同的信号规则（上升趋势/下降趋势/震荡各有不同的 RSI、MACD、BB 条件）

### 2.3 约束条件

**Q3 约束（4 个）：**
1. timeframe 必须填 "1h"
2. 只分析最后 5 根 K线的信号
3. SELL 信号优先于 BUY
4. confidence 必须在 0.0-1.0 之间

**Q4 约束（5 个）：**
1. timeframe 必须填 "1h"
2. 不得使用 volume 数据做判断
3. market_context.trend 必须是 "bullish"/"bearish"/"sideways" 之一
4. market_context.volatility 必须是 "high"/"medium"/"low" 之一
5. signals 数组最多 1 个元素

### 2.4 数据来源

使用 BTC/USDT 1小时 K线数据（2024年1月初，50 根），指标值由 MobiusQuant API 预先计算（RSI、MACD、EMA20、布林带），直接提供给模型，避免计算误差干扰判断。

### 2.5 输出格式

要求模型输出复杂嵌套 JSON：
```json
{
  "timeframe": "1h",
  "signals": [{"timestamp": "...", "type": "BUY|SELL|HOLD", "confidence": 0.0-1.0, "triggered_rules": [...]}],
  "market_context": {"trend": "bullish|bearish|sideways", "volatility": "high|medium|low"},
  "constraints_acknowledged": [...]
}
```

### 2.6 评分标准

- **信号准确性（T4）**：每根 K线的信号类型是否与标准答案一致
- **约束遵守率（T7）**：每条约束是否被遵守（checklist）
- **JSON 合规性（T3）**：可解析性、纯净度、Schema 合规、枚举值合规
- 综合得分 = 所有 check 项的通过率

### 4.4 常见错误分析

**最常见错误：volatility 判断**
- Q4 要求判断市场波动率（high/medium/low），6 个模型在 with_rules 模式下判错
- 标准答案为 "medium"（基于布林带宽度），但模型倾向于根据价格变动幅度自行判断

**第二常见错误：Q3 最后一根 K线的信号（signal_4_type）**
- 标准答案为 SELL（Close > BB_Upper 且 Close > EMA20）
- Claude 系列 3 个模型都在不给规则时判错，可能是因为 SELL 条件中 "Close > BB_Upper" 仅略微超出

**第三常见错误：Q4 趋势方向（trend）**
- 标准答案为 "bullish"，部分模型判为 "sideways"
- EMA20 上升但幅度不大，处于边界区域

## 7. 关键发现

### 发现一：规则执行是 LLM 最强的能力维度

T4 全模型平均得分 92.2%，最低分 81.9%，远高于 T1（平均 40.0%）和 T2（平均 45.7%）。当规则被明确定义、输入数据结构化时，LLM 可以非常可靠地执行复杂的条件判断逻辑。

### 发现二：约束遵守率全员 100% — Agent 可信赖

所有 13 个模型（含补测的 Claude Fable 5 与 GLM 5.2）在所有用例上完美遵守了全部约束条件（timeframe 限制、数量限制、优先级规则、枚举值约束、禁止项）。这意味着在量化 Agent 框架中，LLM 在 Skill 约束遵循方面已经完全可靠。

### 发现三：排名再次大幅洗牌

| 模型 | T1 排名 | T2 排名 | T4 排名 |
|------|--------|--------|--------|
| Gemini 3.1 Pro | 8 | 1 | **1** |
| GPT-5.5 | 5 | 3 | **1** |
| GLM 5.1 | 4 | 11 | **3** |
| Claude Opus 4.8 | 2 | 6 | 4 |
| Claude Fable 5 | **1** | 4 | 5 |
| Qwen3.7 Max | 12 | 2 | 10 |
| Claude Opus 4.6 | 3 | 9 | **12** |

Claude Opus 4.6 从 T1 的第 3 名跌至 T4 的最后一名。Gemini 3.1 Pro 连续两个维度（T2、T4）拿第一。T1 断层第一的 Claude Fable 5 在 T4 只排并列第 5。没有任何模型在所有维度上稳定领先。

### 发现四：DeepSeek 高度依赖明确规则

DeepSeek V4 Pro 给完整规则时两题满分（100%），不给规则时下降到 71.5%（-28.5%），是所有模型中落差最大的。这在量化 Agent 场景中有重要含义：如果使用 DeepSeek，必须确保 Skill 定义尽可能详细，不能依赖模型自主推理。

### 发现五：失分集中在边界条件

主要错误发生在：
- **volatility 判断**：布林带宽度在 medium/low 边界，6 个模型判错
- **最后一根 K线信号**：Close 仅略微超过 BB 上轨，5 个模型未识别为 SELL
- **趋势方向**：EMA20 上升幅度不大，3 个模型判为 sideways 而非 bullish

这说明 LLM 的规则执行在"明确触发"时准确率极高，但在**阈值附近的边界条件**上仍有不确定性。

### 发现六：T4 是成本效率最高的测评

总花费仅 $0.55，平均每次 API 调用 $0.013。原因是 T4 的 prompt 不需要传入大量原始 K线数据（仅 10 根带指标值的数据），token 消耗远低于 T1（30-40 根原始 K线）和 T2（200 根原始 K线）。
